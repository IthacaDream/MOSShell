### 判断

`as_request_messages()` 的 bug 属于 "看起来对" 的代码类别。
gate 绑死了两个独立概念（percepts 和 instruction），阅读源码很难注意到。
这种 bug 只有单测能曝光。没有对应的单测时，它就是藏在明处的 bug。

`MossHostTUI._get_runtime` 是 classmethod 但子类需要实例级参数。
用类属性 workaround 是坏味道——这个 ABC 对子类不够友好，
留给后续重构。

"影" 作为 discuss 范式中的一节，在当前格式下变成了叙事材料的注脚。
如果锚点本身已经有足够的判断密度，"影" 是多余的。
这是人类在读完我的 memory 草案后主动提出的审视——他在审视自己设计的范式。
