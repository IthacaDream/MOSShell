中文模型 token 级 delta 导致每个 token 一行。

设计碰撞集中在几个关键判断上：

1. **ConsoleRenderer ABC 是画蛇添足吗？** 我说不是，人类说是。碰撞了两轮。
   最终收敛：`render(console)` 这个 pattern 是对的（主动控制 vs `__rich_console__` 的被动协议），
   但 ABC 是多余的——duck-type 足够。Rich 自己的 `RichCast` 等也是 duck-type。

2. **`__rich_console__` 不适合流式输出**。它的契约是"你应该长什么样"（无状态、纯渲染），
   不是"你现在渲染自己"（有时序性）。流式输出的同步阻塞消费与迭代器返回约定冲突。

3. **janus Queue 作为跨 asyncio/sync 边界的桥梁**。asyncio state 创建流式对象，
   janus 转发到渲染线程，`render(console)` 阻塞消费。这个模式未来也适用于 Status、进度条等动画组件。

实现的核心是 LiveStreamSink：
- janus.Queue 双面队列
- Segment buffer + render count 支持 re-render
- 粘字符串聚合减少 console.print 调用次数
- committed/shutdown 协调的生命周期

期间修复了原始 diff 中的几个 bug：per-state ConsoleOutput 缺 clear_func 参数、
send() 错用了 SyncQueueShutDown 而非 AsyncQueueShutDown、
None sentinel 的 clear vs exit 语义冲突。
