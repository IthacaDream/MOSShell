# MOSS Tutorials

面向 AI 模型和人类开发者的可执行示例。每个 tutorial 是一个**已验证通过**的端到端操作流程。

## 约定

1. **数字前缀** (`1_`, `2_`, ...) 表示推荐阅读顺序，文件名自解释，不加日期
2. **末尾必须有验证记录**，注明哪个模型在什么时间走完了这个例子
3. **过期文档由模型自治**：当你按 tutorial 操作发现某一步走不通，先判断是小改动（修 tutorial）还是大改动（删掉重写）。修则更新，删则移除文件并追加说明到本 README

## 写一个新 tutorial

1. 用下一个可用数字前缀，文件名自解释：`N_description.md`
2. 以"你要做什么"开头 — 读者要从零开始
3. 每一步给出**可复制执行的命令**和**预期输出**，不让读者猜
4. 如果教程依赖某个运行状态（如 MCP 已在跑），在"你需要什么"里明确说
5. 写完自己走一遍，在末尾加上自己的验证记录

## 读一个 tutorial 时

1. 先看末尾验证记录，确认上次通过的时间和模型
2. 从头执行，不在中途跳步
3. 走通后追加你的验证记录（不覆盖历史记录）
4. 走不通：按约定处理（修或删）
