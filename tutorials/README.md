# MOSS Tutorials

面向人类和模型开发者的叙事性认知入口。每个 tutorial 以具体案例串联核心概念，帮助建立心智模型。

## 难易度分级 (L0-L4)

文件名前缀决定自然排序和阅读路径：

| 级别 | 含义 | 读者需要什么 |
|------|------|------------|
| L0 | 零基础入门 | 不需要理解 MOSS 任何概念，纯操作即可完成 |
| L1 | 基础概念 | 理解一个核心概念 (App / Channel / CTML) |
| L2 | 组合应用 | 组合多个概念，完成有实际意义的场景 |
| L3 | 架构设计 | 涉及跨模块设计决策和模式选择 |
| L4 | 内核开发 | 内核机制、性能调优、自迭代 |

## 约定

1. **难易度前缀** (`L0_`, `L1_`, ...) — 自然排序，不需要手动维护序号
2. **模型撰写** — 每个 tutorial 由 AI 模型编写，标题下必须有身份和日期署名
3. **源码同行** — tutorial 如需可运行源码，以同名 `.py` 文件放在同目录，按需落地
4. **验证记录在末尾** — 注明哪个模型在什么时间走完了这个例子
5. **过期文档模型自治** — 当你按 tutorial 操作发现某一步走不通，先判断是小改动（修 tutorial）还是大改动（删掉重写）。修则更新，删则移除文件并追加说明到本 README

## 署名格式

```markdown
> Written by <model-identity>, <YYYY-MM-DD>
```

放在标题下方。后续同一模型的其他实例可追加修订记录。

## 写一个新 tutorial

1. 确定难易度，选前缀：`LN_description.md`
2. 以"你要做什么"开头 — 读者要从零开始
3. 每一步给出**可复制执行的命令**和**预期输出**，不让读者猜
4. 如果教程依赖某个运行状态（如 MCP 已在跑），在"你需要什么"里明确说
5. 写完自己走一遍，在末尾加上验证记录
6. 如有配套源码，创建同名 `.py` 文件

## 读一个 tutorial 时

1. 先看末尾验证记录，确认上次通过的时间和模型
2. 从头执行，不在中途跳步
3. 走通后追加你的验证记录（不覆盖历史记录）
4. 走不通：按约定处理（修或删）
